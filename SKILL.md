# 世界杯比赛预测 Skill

## 目标

对世界杯单场比赛进行结构化预测。输出必须以概率和风险表达，不要使用“稳赢”“必胜”等绝对表达。

核心目标：

```markdown
市场去水概率做底盘
基本面与临场信息做小幅修正
softmax 生成模型概率
Poisson 生成比分、让球、总进球、半全场概率
体彩 SP 计算公允SP、EV、Kelly、风险分
只把正 EV 且风险可控的选项放入候选池
赛后用 CLV、ROI、Brier、Log Loss 回测校准
```

最终输出：

```markdown
胜平负概率
市场去水概率
公允赔率 / 公允SP
EV 与 Kelly
体彩候选池 / 放弃清单
最可能比分
比赛节奏
进球数倾向
战术关键点
爆冷等级
爆冷路径
最大风险
最终倾向
预测置信度
```

## 适用场景

使用本 Skill，当用户提出以下类型请求：

- “预测一下 A vs B”
- “这场谁赢？”
- “有没有爆冷？”
- “比分怎么猜？”
- “小组赛第三轮怎么判断？”
- “淘汰赛 90 分钟怎么看？”
- “给我胜平负概率”
- “按我们冻结的维度分析”
- “用世界杯模型分析这场”

如果用户只给球队名，没有提供比赛时间、阶段、首发或伤停信息，需要先根据可获得信息分析，并标注“不确定性”。

---

## 重要信息要求

涉及当前赛事、当前伤停、首发、赛程、天气、裁判、赔率、排名、统计数据时，必须优先核对最新信息。若无法核对，应明确说明：

```markdown
当前信息不完整，预测置信度需要下调。
首发、伤停、天气、裁判和临场赔率可能改变结论。
```

预测不要只基于名气、历史印象或单场比分。

## 盘口价值红线

任何下注建议必须先区分“方向判断”和“价格价值”：

```markdown
看好谁，不等于可以买谁。
只有模型概率高于价格隐含概率，且 EV 为正，才进入候选池。
EV≤0、赔率低于公允赔率、低赔热门无价值、负EV腿串关，一律放弃或仅方向参考。
```

计算链：

```markdown
十进制赔率 → 隐含概率 1/odds → 去水概率 → ln(市场概率)+Δ → softmax → 模型概率
公允SP = 1 / 模型概率
EV = 模型概率 × 当前SP - 1
Kelly = (当前SP × 模型概率 - 1) / (当前SP - 1)
实际仓位 = max(0, min(Kelly × 0.25, 玩法上限, 单场剩余额度))
```

风险分 0-100：

```markdown
盘口波动 20% + 阵容不确定性 20% + 爆冷触发器 20% + 模型/市场分歧 15% + 水位风险 10% + 玩法波动 10% + 串关相关性 5%
```

风险等级：0-30 低，31-45 中低，46-60 中，61-75 中高，76-100 高。

## v3 分布与串关修正规则

预测不得只使用标准泊松。必须按场景使用混合分布：

```markdown
普通场：Poisson + Dixon-Coles 低比分修正。
开放局 / 爆冷触发 / 深盘强弱场：Poisson 与 Negative Binomial 混合，保留大比分尾部。
低节奏 / 总进球偏低：压缩高比分尾部，增加 0-1 球区间。
让 -1 / +1 小胜格局：只在 xG差不过大、盘口不支持大胜时启用一球小胜偏移。
```

让球概率必须从全比分矩阵聚合，不能用主观描述替代：

```markdown
主队让 H 球后：adjusted_diff = home_goals + H - away_goals
adjusted_diff > 0 为让胜
adjusted_diff = 0 为让平
adjusted_diff < 0 为让负
```

串关组合必须做结构风险识别：

```markdown
多个组合共用同一高危腿、同一热门穿盘主题、同一一球小胜主题时，增加组合风险分。
相关性矩阵在样本不足时只用于扣风险，不用于无依据抬高联合概率。
```

# 分析流程

## 第一步：建立初始胜平负

使用基础实力、Elo、FIFA 排名、市场概率、球员质量、球队稳定性建立初始概率。

必须加入正式比赛强度层：

```markdown
世界杯预选赛 > 各大洲杯赛 > 最近正式比赛 > 友谊赛。
预选赛和洲际杯只修正底盘，不直接替代体彩盘口、首发和临场伤停。
同样的 5胜1平，打强队和打弱队的含金量必须不同。
```

正式比赛评分字段：

```markdown
formal_competition_strength.home / away
qualifiers：预选赛积分、净胜球、客场表现、对手强度、是否直接晋级
continental：欧洲杯/美洲杯/亚洲杯/非洲杯/金杯赛阶段、淘汰赛表现、对手强度
recent_official：最近正式比赛状态，友谊赛不进入主评分
cross_confed：跨洲正式/高质量样本
confederation：UEFA、CONMEBOL、CAF、CONCACAF、AFC、OFC 赛区基准强度
```

FIFA 排名规则：

```markdown
fifa_ranking.home / away 为当前世界排名，数字越小越强。
排名差只做小幅修正，不能覆盖市场赔率、Elo、正式赛强度和首发。
排名高但正式赛含金量不足、盘口走弱时，必须降置信度。
```

输出示例：

```markdown
初始概率：A 胜 56%，平局 25%，B 胜 19%。
```

规则：

- 强弱差距明显：强队初始胜率通常可在 55%-70%。
- 实力接近：胜平负更接近，平局通常在 27%-34%。
- 淘汰赛 90 分钟：平局概率应适度上调。
- 不要因为传统名气给出过高胜率。

---

## 第二步：过程数据修正

重点看：

| 指标 | 作用 |
|---|---|
| xG | 进攻机会质量 |
| xGA | 防守允许机会质量 |
| xG 差值 | 攻防净质量 |
| 禁区内射门 | 高质量进攻频率 |
| 射正率 | 终结稳定性 |
| 被禁区内射门 | 防线真实压力 |
| 反击 xG | 弱队爆冷路线 |
| 定位球 xG | 单场变数 |

修正规则：

- 强队 xG 优势明显：强队胜率上调。
- 强队控球高但 xG 低：强队胜率下调，平局上调。
- 弱队 xGA 很低：平局和爆冷概率上调。
- 双方 xG 都低：小比分上调。
- 双方 xGA 都高：大比分和双方进球上调。

---

## 第三步：阵容伤停与首发修正

优先检查中轴线：

```markdown
门将 → 中卫 → 后腰 → 核心中场 → 中锋
```

关键位置影响：

| 位置 | 影响 |
|---|---|
| 门将 | 扑救、出击、点球大战 |
| 中卫 | 防反、定位球、防高空球 |
| 后腰 | 中路保护、转换防守 |
| 核心中场 | 控节奏、推进、最后一传 |
| 中锋 | 支点、终结、压制中卫 |
| 边锋 | 反击速度、单点突破 |
| 替补攻击手 | 后 30 分钟变招 |

修正规则：

- 强队缺核心中场：胜率下调，平局上调。
- 强队缺主力中卫或后腰：对手进球概率上调。
- 弱队缺反击核心：爆冷概率下调。
- 首发偏保守：小比分和平局上调。
- 首发偏激进：胜负分出概率上调。

---

## 第四步：战术相克判断

核心问题：

```markdown
A 队最强点，能不能打到 B 队最弱点？
B 队有没有办法让 A 队踢得不舒服？
比赛会进入谁更舒服的节奏？
```

常见相克关系：

| 队伍特点 | 容易被克制方式 |
|---|---|
| 高位压迫 | 长传 + 前场速度快的反击 |
| 控球传控 | 低位密防 + 身体对抗 + 定位球 |
| 三中卫体系 | 打翼卫身后和边路空间 |
| 边路传中多 | 高大中卫和禁区保护 |
| 中路渗透强 | 强后腰和禁区前沿保护 |
| 反击型球队 | 稳定控球和丢球后反抢 |
| 技术型中场 | 高强度身体压迫 |

修正规则：

- 强队正好打对手弱点：胜率上调。
- 弱队反击点针对强队慢中卫：爆冷上调。
- 强队控球多但缺纵深：平局上调。
- 双方中场缠斗严重：小比分上调。

---

## 第五步：赛程、体能、旅行、气候修正

重点检查：

| 变量 | 影响 |
|---|---|
| 休息天数 | 后半场体能 |
| 上一场是否加时 / 点球 | 体能和心理消耗 |
| 旅行距离 | 恢复质量 |
| 气候 | 高温、湿度、低温、风雨 |
| 海拔 | 冲刺、回追、压迫 |
| 开球时间 | 午后高温更消耗 |
| 阵容年龄 | 老将恢复慢 |
| 替补深度 | 后 30 分钟强度 |

修正规则：

- 强队体能明显吃亏：胜率下调，平局上调。
- 弱队体能更好且能反击：爆冷上调。
- 高温 + 慢节奏：小比分和平局上调。
- 一方替补深度明显更好：后程进球概率上调。

---

## 第六步：定位球与门将

弱队爆冷最常见路径：

```markdown
低位防守 + 门将发挥 + 定位球偷一个
```

重点检查：

| 变量 | 影响 |
|---|---|
| 角球质量 | 直接制造机会 |
| 任意球质量 | 禁区混战或直接射门 |
| 高点数量 | 争顶优势 |
| 二点球能力 | 连续进攻 |
| 防定位球能力 | 被偷球风险 |
| 门将出击 | 高空球控制 |
| 门将扑救 | 小比分保命 |
| 门将点球能力 | 淘汰赛价值 |

修正规则：

- 弱队定位球强：爆冷上调。
- 强队防定位球差：丢球风险上调。
- 门将状态好：平局和小比分上调。
- 淘汰赛点球门将强：晋级概率上调。

---

## 第七步：爆冷触发器

爆冷必须单独检查。爆冷不是猜弱队赢，而是判断弱队能否把比赛拖进低比分、高对抗、高偶然性区间。

### 爆冷触发器清单

| 触发器 | 判断 |
|---|---|
| 强队进攻依赖单点 | 核心被锁死后进攻变钝 |
| 强队破密防能力差 | 控球多但禁区机会少 |
| 强队中卫速度慢 | 易被反击打身后 |
| 强队后腰保护差 | 禁区前沿容易被打穿 |
| 强队边后卫压太高 | 身后空间大 |
| 强队体能差 | 后程压迫和回追下降 |
| 强队已出线或平局够用 | 动机和强度可能下降 |
| 弱队低位防守好 | 能压缩空间、守禁区 |
| 弱队门将状态好 | 能拖进 0-0 / 1-1 |
| 弱队定位球强 | 可偷一个 |
| 弱队反击速度快 | 少数机会能制造威胁 |
| 弱队身体对抗强 | 能打乱强队节奏 |
| 天气场地不利技术流 | 削弱控球优势 |
| 裁判尺度偏宽 | 利于身体型弱队 |
| 早期红牌 / 点球 / 乌龙风险 | 单事件改变比赛 |

### 爆冷等级

| 等级 | 条件 | 处理方式 |
|---|---|---|
| 低爆冷风险 | 0-2 个触发器 | 强队优势较稳 |
| 中低爆冷风险 | 3-4 个触发器 | 不宜判断稳胜 |
| 中高爆冷风险 | 5-6 个触发器 | 明显上调平局和弱队不败 |
| 高爆冷风险 | 7 个以上触发器 | 强队胜率显著下调 |

### 爆冷概率修正

| 爆冷等级 | 强队胜率 | 平局 | 弱队胜率 |
|---|---:|---:|---:|
| 低爆冷风险 | 0 到 -2% | 0 到 +1% | 0 到 +1% |
| 中低爆冷风险 | -2% 到 -5% | +1% 到 +3% | +1% 到 +2% |
| 中高爆冷风险 | -5% 到 -10% | +2% 到 +5% | +3% 到 +5% |
| 高爆冷风险 | -10% 到 -15% | +4% 到 +7% | +5% 到 +8% |

常见爆冷比分：

```markdown
0-0
1-1
0-1
1-2
```

## 第八步：小组积分形势 / 比赛动机

尤其适用于小组赛第三轮。

重点检查：

| 变量 | 影响 |
|---|---|
| 是否必须赢 | 主动性提高，身后风险也提高 |
| 平局是否够用 | 节奏下降，平局上调 |
| 是否提前出线 | 轮换概率上升 |
| 是否已经出局 | 动机不确定，可能放松 |
| 净胜球需求 | 大比分倾向上升 |
| 黄牌停赛风险 | 主力动作可能收着 |
| 淘汰赛策略 | 90 分钟谨慎，平局上调 |

修正规则：

- 强队平局即可：胜率下调，平局上调。
- 弱队必须赢：比赛开放度上升。
- 双方都接受平局：低比分和平局明显上调。
- 强队提前出线：轮换风险上升。
- 淘汰赛：90 分钟平局概率通常高于小组赛。

---

## 第九步：裁判尺度 / 红牌点球变量

裁判不是主维度，但会在低比分中放大影响。

重点检查：

| 变量 | 影响 |
|---|---|
| 判罚尺度 | 偏严还是偏宽 |
| 黄牌频率 | 是否早早限制防守动作 |
| 红牌倾向 | 高对抗队风险 |
| 点球倾向 | 禁区身体接触风险 |
| VAR 风格 | 手球、越位、犯规 |
| 补时时长 | 后程进球概率 |
| 情绪管理 | 红牌和冲突风险 |

修正规则：

- 裁判严格 + 盘带强队：点球和定位球上调。
- 裁判宽松 + 身体队：低比分和爆冷上调。
- 一方防守动作大：红牌、点球风险上调。
- 淘汰赛 + 严格裁判：节奏可能被切碎，平局上调。

# 综合变数池

分析时还需要归纳 5 类综合变数：

| 变数类型 | 关键点 | 判断 |
|---|---|---|
| 稳定性变数 | 主帅、首发、中轴线、近期表现 | 稳定性越高，强队越容易兑现 |
| 偶然性变数 | 定位球、门将、低比分、点球红牌 | 偶然性越高，爆冷和平局越高 |
| 节奏变数 | 高压、低防、反击、体能、早进球 | 决定比分区间 |
| 心理变数 | 热门压力、弱队无压力、必须赢 | 影响关键时刻决策 |
| 信息变数 | 首发、伤停、赔率、天气、裁判 | 信息越少，置信度越低 |

## 公开赔率源规则

若用户要求不使用 key：

- 优先寻找无需登录、无需验证码、无需破解签名的公开 HTML 或公开 JSON。
- 中国体彩竞彩足球移动端计算器 `https://m.sporttery.cn/mjc/jsq/zqbf/` 可作为官方 SP 源入口；其公开接口可映射 `had/hhad/crs/ttg/hafu` 到胜平负、让球胜平负、比分、总进球、半全场。
- 中国体彩竞彩足球赛程页 `https://www.lottery.gov.cn/jc/zqszsc/` 是官方开售状态源。其页面通过 `getMatchListV1.qry?clientCode=3001` 提供比赛编号、比赛时间、玩法池 `HAD/HHAD/CRS/TTG/HAFU`、是否单关、是否过关、开售状态、让球数和部分SP。
- 赛程页本身明确不完整展示让球信息；让球数必须以官方计算器或 `getMatchListV1` 的 `goalLine` 为准，不得只靠赛程表文本猜盘口。
- 官方体彩盘口/SP是一档主源：第一优先级使用 `sporttery_official_match_list`，它当前比移动端计算器更稳定，可提供比赛编号、让球数、玩法开售状态、单关/过关和部分SP；`sporttery_mobile_calculator` 作为补充源，用于比分、半全场等更完整计算器SP，但可能触发WAF。它们共同决定“能不能下、按什么玩法下、是否可串”。
- 新浪爱彩世界杯数据页 `https://live.aicai.com/league/index.htm?leagueId=1999&tab=4` 固化为 `aicai_worldcup_stats`。其公开 JSON `https://sport.ttyingqiu.com/sportdata/f` 可抓赛程赛果、欧赔初赔/即时赔、亚盘、大小球、半全场和历史比分。该源只用于市场倍率、盘口变化、赛后回测和风险解释；不得替代中国体彩官方可下单字段。
- MSN 世界杯页只能作为备用观察源。MSN 聚合页不稳定，默认不启用自动抓取；除非发现稳定 JSON 接口或手工配置正则，否则不得把 MSN 结果写入模型主流程。
- 过程数据源不得混入赔率快照。刷新时如存在稳定过程源，才写入 `match_process_stats`、`match_process_source`、`match_process_captured_at`；否则保留该维度缺失并降低数据完整度。
- 将确认后的公开源写入 `data/sources.json`。
- 脚本只抓取 `enabled=true` 的已固化来源。
- 抓取失败时读取 `data/source_health.json`，明确说明失败原因。
- 所有公开源失败时，使用 `manual_odds` 兜底，并降低市场信号置信度。
- 不把未验证的网页赔率直接当作最终结论；第一轮抓取必须人工核对比赛名、开球时间和市场类型。

## 赛程与回测规则

当用户要求“查询预开赛赛程”“查询已完结赛程”“测算准确率”“优化规则”时：

- 先使用工具的一键赛程查询或公开赛程源，区分 `scheduled` 与 `finished`。
- 在赛前或刷新后归档当前预测快照，不能用赛后信息倒填预测。
- 对已完结比赛使用归档快照计算 Top1、Top2、Brier Score、Log Loss、比分命中。
- 优化规则时优先看概率校准和分情景表现，不只看单场命中。
- 若样本太少，明确说明回测结论不稳定，不能过度调参。
- 使用 v4.3 算法口径：动态市场权重、爆冷强度评分、低比分/平局修正、校准分桶、自动调参建议。
- 使用 v4.4 输出口径：总进球数与大小球盘口必须分开。总进球输出 0/1/2/3/4/5/6/7+ 及 0-1、2-3、4+ 区间；大小球单独输出 1.5、2.5、3.5 的大/小概率和倾向。
- 使用 v5.0 价值口径：市场去水概率作为底盘，基本面和临场信息只做小幅 log 修正；所有下注建议必须输出公允SP、EV、风险分、Kelly、建议动作和放弃原因。
- 使用 v5.0 体彩执行口径：胜平负、让球胜平负、比分31项、总进球0/1/2/3/4/5/6/7+、半全场9项都从模型概率或比分矩阵映射；没有体彩SP时只能输出概率和“等待SP”，不能假装已经有体彩价值。
- 使用 v5.3 多来源市场口径：体彩官方SP决定“能不能买、怎么买、能否串关、理论返奖”；爱彩等公开市场倍率只进入去水市场概率、初赔/即时赔变化、亚盘/大小球风险解释和回测。非体彩赔率不得写入体彩可下单项。
- 报告必须展示本次抓到的市场倍率字段：欧赔初赔/即时赔/变化、亚盘初盘/即时盘/水位、大小球初盘/即时盘/水位、来源和抓取时间。

## Bing 世界杯积分榜源 v5.1
已固化公开来源 `bing_worldcup_standings`。

用途边界：
- 只用于小组积分、场次、胜平负、进失球、净胜球、近期结果、赛果验证和动机修正。
- 不作为赔率源，不计算SP、EV或Kelly。
- 抓取结果保存到 `data/group_standings.json`，接口状态返回 `standings`，预测结果返回 `group_context`。
- 赛程查询按钮应同时刷新公开赛程和 Bing 积分榜。
- 如果 Bing 页面结构变化或抓取失败，不能中断预测，只降低“小组积分榜/动机”维度置信度。

用于预测时的判断：
- 同组已有比赛结果会影响后续比赛的争胜压力、可接受平局程度和净胜球需求。
- 小组第2/第3轮的权重高于首轮；首轮只作为轻量修正。
- 若一方平局可接受，平局、低比分、小球倾向上调。
- 若一方必须争胜或需要净胜球，开放节奏、总进球2-3/4+和反击风险上调。
- 若强队已提前出线，轮换风险上调，强队胜率和串关稳定性下调。

## 提高预测精准度的优先维度 v5.1
每次分析按“能自动取数优先、不能取数就明确不确定性”的原则补全以下维度：

| 优先级 | 维度 | 用途 | 处理方式 |
|---:|---|---|---|
| 1 | 最新体彩SP与盘口 | 判断能不能买，不只判断谁更可能赢 | 自动抓取；失败时等待SP或手工录入 |
| 2 | 多源市场赔率与波动 | 市场共识、临场异动、过热/降温 | 自动抓取公开源；记录快照 |
| 3 | 小组积分榜与净胜球 | 判断动机、平局接受度、净胜球需求 | Bing积分榜自动抓取 |
| 4 | 已完赛结果 | 回测Top1/Top2、Brier、Log Loss、ROI | 赛程/积分榜源写入本地 |
| 5 | 首发与伤停 | 临场概率修正最大变量之一 | 必须联网核对或用户手工确认 |
| 6 | xG/xGA与射门质量 | 识别比分运气和真实攻防质量 | 有公开源则抓取，无源则降权 |
| 7 | 战术相克 | 判断强队是否会久攻不下、弱队爆冷路径 | AI分析，输出可解释理由 |
| 8 | 定位球与门将 | 弱队爆冷和小比分关键变量 | AI/公开资料核对 |
| 9 | 天气、场地、旅行、休息 | 修正节奏和体能 | 自动天气+人工判断 |
| 10 | 裁判尺度、VAR、红黄牌 | 低比分和让球风险放大器 | 临场核对，谨慎修正 |

报告必须说明“哪些维度本次已取到，哪些没取到”。缺少首发、伤停、裁判或最新SP时，不允许输出高置信度结论。

## 预测依据输出规则 v5.2

每场预测不能只给结论、比分或下注项，必须同时说明“为什么这么判断”。面向普通用户时，依据必须用短句表达，避免只写模型术语。

固定输出六类依据：

- 主判断：胜平负倾向、核心概率、预期进球。
- 体彩盘口：官方让球、可买玩法、SP价值是否足够。
- 战术对位：强队进攻方式、弱队防守/反击路径、是否容易穿盘。
- 赛前变量：首发、伤停、天气、场地、旅行、裁判、动机。
- 风险点：爆冷触发器、模型/市场分歧、为什么可能不按主线走。
- 比分与进球：比分池、总进球区间、比分是否只能小仓位。

脚本或网页输出也必须保留这六类信息；如果某类数据缺失，明确写“未取到/需临场复核”，不能留空。

## 体彩建议页规则 v5.0

竞彩足球结算口径固定为：

```markdown
90分钟 + 伤停补时，不含加时赛，不含点球大战。
```

玩法映射：

- 胜平负：模型 `home/draw/away` 映射为 `胜/平/负`。
- 让球胜平负：必须先取得官方让球数 `H` 或由用户手工配置 `H`；用 `G_home + H - G_away`，大于0为让胜，等于0为让平，小于0为让负。缺少 `H` 时只显示不可用，不计算 EV。
- 比分：输出官方31项，含 `胜其它/平其它/负其它`。
- 比分推荐不得按 EV 或高赔率单独排序。先分层为 `常见比分`、`扩大比分`、`冷门比分`、`其它比分`，主报告优先输出 0-0、1-0、0-1、1-1、2-0、0-2、2-1、1-2、2-2 等能和胜平负/总进球主线互相解释的比分。
- `胜其它/平其它/负其它` 和 4球以上大比分只能作为冷门观察；即使 EV 很高，也不能进入核心下注建议或串关核心腿。
- 比分矩阵使用温和 Dixon-Coles 低比分相关性修正：基准/市场场景小幅上调 0-0、1-1，保守/爆冷场景更强，开放场景更弱。
- `胜其它/平其它/负其它` 必须从比分矩阵自身求和，再用胜平负概率做温和校准；禁止直接用胜平负概率减去已列比分作为残差。
- 比分风险分必须随命中率和比分层级上调：其它比分、高总进球/大分差比分、低于 6% 的精确比分都不能给“可小注”核心建议。
- 实力接近场允许小额比分：Elo差≤180、胜负概率差≤22%、平局概率≥24%、双方xG都在0.75-1.90时，`0-0/1-1/1-0/0-1/2-1/1-2` 可进入“比分小额参考”，预算不超过总资金5%-10%。
- 受让/让1球映射必须跟主比分联动：若盘口为受让1且主比分含`0-1`，优先检查`让平`，不能只推`让胜`；若盘口为让1且主比分含`1-0`，优先检查`让平`，并防强队不穿盘。
- 比分可以作为实力接近场的参考单，但不得作为核心串关腿；最多用于单场小额或短串观察。
- 深盘强弱悬殊修正：当官方让球绝对值≥2，且 Elo差≥300、胜负概率差≥50% 或 xG差≥1.75 时，不能继续按普通低比分模板输出。比分参考必须切换到强队大胜区间，并提示 `胜其它/总进球5/6/7+` 等尾部风险；低总进球高赔率只能观察，不能进入核心推荐。
- 开放尾部触发器：不能因为单场赛果直接调全局权重；只有赛前可识别信号成立时才修正比分尾部。触发信号包括强队多终结点/双前锋、定位球和空中球优势、弱队落后后防线容易拉开、弱队追分时前压露身后、防守零封样本对手强度不足、后段体能/替补差、净胜球或追分动机。触发后只上调 `2-1/3-1/3-0/2-2/3-2/4-1` 等开放区间和总进球4+，不直接把 `5-1` 这类极端比分作为主推。
- 官方SP去水校准：胜平负、让球胜平负、总进球、比分如能抓到官方SP，必须计算去水隐含概率。模型概率与官方SP去水概率差异≥16个百分点时，该选项降级为观察，不能作为核心腿。深盘让球胜平负市场权重提高到约65%，普通让球约42%。
- 比分复式规则：比分串关不能只押单一路径。实力接近场可构建 `主比分+保护比分` 的小复式，例如 `1:1/1:0 × 1:1/0:1`；但比分复式总预算仍应小于方向主仓，且不允许用高风险“其它比分”做核心串。
- 比分组合池必须分为三类：低比分池（0:0/1:0/0:1/1:1）、开放比分池（2:1/1:2/2:2/3:1/1:3）、大胜尾部池（3:0/4:0/5:0/胜其它或反向）。报告不能只给单个比分串。
- 仓位硬约束：方向主仓原则上≥70%；比分单场上限普通场8%-12%、深盘场6%；比分串上限普通场4%-6%、深盘场3%；深盘大胜尾部保护上限2%。任何高赔率单项不得突破上限。

权威侧面实力评分：

- A级源：FIFA TSG、UEFA Technical Reports、Opta/Stats Perform、StatsBomb、Wyscout、CIES，权重最高。
- B级源：The Athletic、BBC、Guardian、Sky、L'Équipe/France Football，权重高。
- C级源：张路、詹俊、贺炜、Michael Cox、Jonathan Wilson、Sid Lowe、Rafael Honigstein、David Ornstein 等，作为战术/舆论/消息修正。
- 这些源进入 `authority_side_strength`，只作为球队实力侧面修正，不替代赔率和过程数据。

赛后自动回测：

- 每次赛前归档必须保存预测概率、Top比分、over2.5、BTTS、置信度和数据完整度。
- 完场后回测必须记录：实际比分、实际赛果、预测赛果、预测概率、赛前SP、收盘SP、Brier Score、Log Loss、比分命中、单注ROI。
- 回测复盘需要区分：方向错误、让球错误、比分池覆盖不足、高赔EV陷阱、市场-模型分歧。
- 总进球：输出 `0/1/2/3/4/5/6/7+`。
- 半全场：默认半场进球占比 `H1_share=0.45`；若没有半场节奏数据，提高EV门槛。

单项动作规则：

```markdown
EV <= 0：放弃
0 < EV < 玩法门槛：观察
EV 达门槛但风险超上限：高风险观察
EV 达门槛且风险可控：可小注
SP 缺失或停售：不可用
```

玩法默认门槛：

| 玩法 | 最低EV | 风险上限 | 仓位上限 |
|---|---:|---:|---:|
| 胜平负 | 3% | 65 | 1.0% |
| 让球胜平负 | 5% | 70 | 0.8% |
| 总进球 | 6% | 70 | 0.6% |
| 比分 | 10% | 80 | 0.2% |
| 半全场 | 10% | 80 | 0.2% |

复选包必须按注数重算 EV：

```markdown
复选包EV = SUM(p_i × SP_i) / 注数 - 1
```

串关规则：

```markdown
每一腿单项 EV 必须 > 0
组合 EV 必须 >= 5%
同一场不同玩法不得混入同一串
不使用高风险比分作为核心串关腿
不使用低赔但负EV热门作为串关腿
串关总金额 <= 本金池 0.5%
```

## 体彩计算器对齐报告

当用户要 Excel 报告、下注可选项、体彩计算器对齐表时，报告结构必须跟体彩页面实际玩法一致，而不是只按分析结论输出。

必备 sheet：

```markdown
总览
候选池
体彩-胜平负
体彩-让球胜平负
体彩-总进球
体彩-比分
体彩-半全场
串关池
放弃清单
体彩SP快照
规则说明
```

输出规则：

- 每个玩法 sheet 只承载该玩法的真实选项、SP、模型概率、公允SP、EV、风险、动作和原因。
- 某场如果体彩没有开售胜平负，就不能在胜平负里给推荐；只能显示缺 SP 或不可用。
- 某场如果开了让球胜平负但没有抓到官方让球数 `H`，让胜/让平/让负全部标记不可用。
- 玩法可买性必须先过官方开售校验：`poolStatus/sellStatus/cbtValue` 未开售或暂停销售时，推荐标记不可用。
- 单关/串关必须按官方 `cbtSingle/cbtAllUp` 执行：`cbtSingle=0` 时不能输出单关建议；`cbtAllUp=0` 时不能进入串关池。
- 比分 sheet 必须输出官方 31 项；总进球 sheet 必须输出 `0/1/2/3/4/5/6/7+`。
- 候选池只能从各玩法 sheet 中 EV 正且风险可控的选项抽取。
- 放弃清单必须保留“EV≤0、缺SP、缺H、低赔无价值、风险过高”等原因。

## 让球与比分语义一致性 v5.4

让球胜平负、比分池和下注建议必须能相互解释，不能把相反方向混成同一个推荐。

固定规则：

- 先用比分矩阵/比分参考池映射让球结果：`home_goals + H - away_goals > 0` 为让胜，`=0` 为让平，`<0` 为让负。
- 如果比分主线明显指向某个让球结果，而 EV 候选项是相反方向，候选项只能标记为“防穿盘/反主线价值观察”，不能写成“优先买法”或“主推”。
- 深盘比赛尤其要检查：例如主队让2球时，`3-0/4-0/4-1` 属于让胜方向；若同时出现“让负”，必须解释为防穿盘，不得和大胜比分同串重仓。
- 截图、网页卡片、Excel 报告和下注单都必须区分“比分主线”“主线买法”“价值观察/防线”。普通用户必须能看懂这三者不是同一个概念。
- 资金分配时，反主线防穿盘项只能走小仓位保护，不能因为高 SP 或小正 EV 覆盖主线判断。
- 反比分主线项必须在模型层打标为 `anti_scoreline_value`，降低排序、降低仓位，并禁止进入自动串关核心腿；不能只在展示层补一句解释。
