# 世界杯比赛预测模型｜冻结维度升级版 v2.0

这套模型作为固定预测框架使用。核心逻辑是：

> **先用基础实力定底盘，再用数据、阵容、战术、体能、动机和爆冷触发器修正概率。最终输出不要只说谁赢，而是输出：胜平负概率 + 最可能比分 + 爆冷路径 + 风险提示 + 最终倾向。**

---

## 一、总维度与权重

| 序号 | 预测维度 | 权重 | 作用 |
|---:|---|---:|---|
| 1 | **基础实力 / Elo / 市场概率** | **18%** | 决定预测底盘，避免完全凭感觉 |
| 2 | **FIFA 世界排名差** | **6%** | 独立校验国家队长期层级 |
| 3 | **世界杯预选赛 / 洲际杯正式赛** | **10%** | 判断正式比赛含金量，区分“真强”和“样本吃红利” |
| 4 | **xG、xGA、射门质量等过程数据** | **16%** | 判断球队真实攻防质量，而不是只看比分 |
| 5 | **权威侧面实力评分** | **8%** | 用 FIFA/UEFA/Opta/StatsBomb/CIES/专业媒体侧面校验 |
| 6 | **赛中/赛后过程表现** | **8%** | 用已完赛统计回测球队真实状态 |
| 7 | **阵容伤停与首发质量** | **12%** | 判断关键球员、核心位置、替补深度 |
| 8 | **战术相克与比赛形态** | **12%** | 判断谁能把比赛带进自己舒服的节奏 |
| 9 | **赛程、体能、旅行、气候** | **7%** | 判断消耗、恢复、环境适应 |
| 10 | **定位球与门将** | **8%** | 世界杯单场里非常容易改变走势 |
| 11 | **爆冷 / 动机 / 裁判变量** | **13%** | 判断弱队不败、积分形势、红牌点球等极端事件 |

# 二、详细维度设计

## 1. 基础实力 / Elo / 市场概率

### 目的

先给比赛定一个基础胜率底盘。强队不是一定赢，但基础实力决定了它在大多数比赛情境下的下限。

### 主要变量

| 子变量 | 观察点 |
|---|---|
| **Elo / FIFA 排名 / 综合评级** | 两队长期实力差距，不能只看最近一两场 |
| **市场隐含概率** | 赔率或市场倾向通常聚合了大量信息 |
| **球员整体质量** | 五大联赛球员数量、欧冠级别球员、核心球星质量 |
| **关键位置质量** | 门将、中卫、后腰、核心中场、中锋、边锋 |
| **大赛经验** | 世界杯、洲际杯、淘汰赛经验 |
| **球队稳定性** | 主帅执教时间、阵容框架是否稳定 |
| **强强对话能力** | 面对高水平对手时是否还能稳定输出 |
| **打弱队能力** | 是否经常控球很多但破密防困难 |

### 加分信号

| 信号 | 说明 |
|---|---|
| 综合实力明显高一档 | 基础胜率可明显上调 |
| 核心球员处于黄金期 | 尤其中前场创造力强 |
| 阵容结构完整 | 有门将、中卫、后腰、创造者、终结者 |
| 市场支持稳定 | 不是突然虚高，而是持续被看好 |
| 大赛经验丰富 | 落后、被压制、加时等场景更稳 |

### 扣分信号

| 信号 | 说明 |
|---|---|
| 名气大于实力 | 传统强队但阵容老化或状态差 |
| 排名虚高 | 预选赛对手较弱，缺乏高质量检验 |
| 依赖单个球星 | 核心被限制后进攻瘫痪 |
| 中后场结构有缺陷 | 强队也可能被弱队反击打穿 |
| 市场过热 | 大众情绪过强，实际优势没那么大 |

### 判断方式

| 判断 | 概率影响 |
|---|---|
| 强队基础实力明显领先 | 强队胜率上调 |
| 双方实力接近 | 平局概率上调 |
| 强队名气大但结构问题明显 | 胜率下调，爆冷概率上调 |
| 弱队整体实力差但有明确武器 | 不一定上调胜率，但要上调爆冷可能 |

## 2. 世界杯预选赛 / 洲际杯正式赛

### 目的

正式比赛比友谊赛更能说明球队真实强度。世界杯预选赛和各大洲杯赛能体现抗压、客场、强强对话、淘汰赛经验和阵容稳定性，是球队实力底盘的重要修正层。

### 数据优先级

| 层级 | 权重含义 |
|---|---|
| 世界杯预选赛 | 最高。看积分、净胜球、客场、对手强度、是否直接晋级 |
| 各大洲杯赛 | 高。看欧洲杯/美洲杯/亚洲杯/非洲杯/金杯赛阶段和淘汰赛质量 |
| 最近正式比赛 | 中高。只看正式比赛状态，友谊赛单独降权 |
| 跨洲高质量样本 | 中。判断赛区强度差和风格适应 |
| 友谊赛 | 低。只做阵容试验和状态参考，不进入主评分 |

### 计算字段

```json
{
  "formal_competition_strength": {
    "confidence": 0.8,
    "home": {
      "confederation": "UEFA",
      "qualifiers": {"points_per_game": 2.2, "goal_diff_per_game": 1.4, "opponent_strength": 0.8},
      "continental": {"stage": "semifinal", "opponent_strength": 0.9},
      "recent_official": {"points_per_game": 2.0, "sample_size": 8}
    },
    "away": {
      "confederation": "CAF",
      "qualifiers": {"points_per_game": 1.7, "goal_diff_per_game": 0.4, "opponent_strength": 0.35}
    }
  }
}
```

### 修正规则

| 情况 | 处理 |
|---|---|
| 正式赛评分明显支持强队 | 强队胜率和 xG 小幅上调，平局小幅下调 |
| Elo 强但正式赛含金量弱 | 降低强队置信度，比分不追极端大胜 |
| 弱队正式赛防守质量高 | 提高受让、低比分和爆冷路径权重 |
| 跨洲样本不足 | 评分 confidence 下调，不能强行放大 |
| 正式赛评分与盘口相反 | 视为风险提示，不直接反着买 |

## 2.5 FIFA 世界排名差

### 目的

FIFA 世界排名用于校验国家队长期层级，解决“只看近期赔率或单场状态导致低估强队底盘”的问题。

### 使用边界

| 规则 | 说明 |
|---|---|
| 数字越小越强 | rank 2 强于 rank 70 |
| 小幅修正 | 只修正基础实力和 xG，不覆盖赔率、Elo、正式赛强度 |
| 防止重复加权 | Elo、市场赔率、正式赛成绩已经体现强弱，FIFA 排名不能再满权重叠加 |
| 排名虚高降权 | 若正式赛含金量弱、盘口走弱或首发不佳，降低 ranking confidence |

### 字段

```json
{
  "fifa_ranking": {
    "home": 2,
    "away": 72,
    "confidence": 0.82,
    "source": "fifa_ranking_seed_2026_06"
  }
}
```

输出字段：

```text
value_model.fifa_ranking_score
value_model.lambda_adjustments.fifa_ranking
dimension_scores[fifa_ranking]
```

## 3. xG、xGA、射门质量等过程数据

### 目的

比分容易骗人，过程数据更能反映球队真实状态。一支队连续赢球但每场 xG 很低，可能只是运气好；一支队连续不胜但 xG 很高，可能只是终结效率暂时差。

### 主要变量

| 子变量 | 观察点 |
|---|---|
| **xG** | 进攻创造出的高质量机会 |
| **xGA** | 防守允许对手创造的机会质量 |
| **xG 差值** | xG - xGA，衡量攻防净质量 |
| **射门次数** | 进攻频率 |
| **禁区内射门** | 比外围远射更有价值 |
| **绝佳机会数量** | 单刀、近距离射门、空门等 |
| **射正率** | 终结稳定性 |
| **被射门次数** | 防线承压程度 |
| **被禁区内射门次数** | 防守质量的重要信号 |
| **高位夺回次数** | 压迫能力 |
| **反击 xG** | 反击质量 |
| **定位球 xG** | 角球、任意球制造威胁能力 |
| **转换进攻质量** | 由守转攻时是否能迅速形成威胁 |

### 数据权重区分

| 类型 | 权重 |
|---|---:|
| 世界杯正赛 | 最高 |
| 洲际杯 / 预选赛关键战 | 很高 |
| 友谊赛 | 较低 |
| 对弱队刷出来的数据 | 要打折 |
| 对强队仍然能打出高 xG | 价值很高 |
| 大轮换比赛的数据 | 谨慎参考 |

### 加分信号

| 信号 | 说明 |
|---|---|
| xG 长期高于对手 | 进攻创造力稳定 |
| xGA 长期较低 | 防守质量稳定 |
| 射门位置好 | 不是靠远射堆数据 |
| 禁区触球多 | 能持续进入危险区域 |
| 输球但 xG 不差 | 可能只是结果运气差 |
| 赢球同时 xG 也好 | 胜利质量高 |

### 扣分信号

| 信号 | 说明 |
|---|---|
| 连胜但 xG 很低 | 可能不可持续 |
| 进球远高于 xG | 终结效率可能回落 |
| xGA 高但没丢球 | 门将或运气掩盖防守问题 |
| 射门多但质量差 | 控球无效，破密防能力不足 |
| 依赖远射 | 面对纪律性防守容易哑火 |
| 被对手频繁打反击 | 淘汰赛风险很大 |

### 判断方式

| 判断 | 概率影响 |
|---|---|
| 强队 xG 优势明显 | 胜率上调 |
| 强队控球高但 xG 低 | 胜率下调，平局上调 |
| 弱队 xGA 很低 | 爆冷和逼平概率上调 |
| 双方 xG 都低 | 0-0、1-1、1-0 概率上调 |
| 双方 xGA 都高 | 2-1、2-2、3-1 概率上调 |

## 3. 阵容伤停与首发质量

### 目的

世界杯预测不能只看国家队名气，必须看当场能上谁。尤其是首发公布后，预测要重新修正。

### 关键位置权重

| 位置 | 重要性 | 影响 |
|---|---:|---|
| **门将** | 极高 | 单场扑救、点球大战、出击失误都会改变结果 |
| **中卫组合** | 极高 | 防反、定位球、防高空球核心 |
| **后腰** | 极高 | 决定防守屏障和攻防转换 |
| **核心中场** | 极高 | 控节奏、推进、最后一传 |
| **中锋** | 高 | 决定终结质量和支点作用 |
| **边锋 / 边翼卫** | 高 | 决定反击速度和边路爆点 |
| **边后卫** | 中高 | 防守边路、传中、套上 |
| **替补前锋** | 中 | 决定后 30 分钟变化 |
| **点球手** | 特殊重要 | 淘汰赛、低比分比赛里影响大 |

### 主要变量

| 子变量 | 观察点 |
|---|---|
| **伤病名单** | 是否缺少绝对主力 |
| **停赛情况** | 黄牌累计、红牌停赛 |
| **首发完整度** | 主力框架是否齐整 |
| **中轴线完整度** | 门将-中卫-后腰-核心中场-中锋 |
| **阵容默契** | 是否长期一起踢 |
| **替补深度** | 落后时有没有变招 |
| **球员疲劳** | 连续首发、俱乐部赛季消耗 |
| **球员状态** | 近期进球、助攻、比赛时间、伤愈复出 |
| **位置客串** | 中卫踢边卫、后腰踢中卫等会增加风险 |
| **点球与任意球人选** | 关键球能力 |

### 加分信号

| 信号 | 说明 |
|---|---|
| 中轴线完整 | 球队稳定性高 |
| 首发连续性强 | 配合成熟，不容易乱 |
| 替补能改变比赛 | 落后时还有办法 |
| 门将状态好 | 低比分比赛价值极大 |
| 核心球员健康 | 进攻组织和终结更稳定 |

### 扣分信号

| 信号 | 说明 |
|---|---|
| 主力门将缺阵 | 防守稳定性下降 |
| 中卫组合临时拼凑 | 防反和定位球风险上升 |
| 后腰缺阵 | 中路保护严重下降 |
| 核心组织者缺阵 | 控球可能变成无效控球 |
| 前锋伤愈强行复出 | 名气大但实际输出可能不足 |
| 替补深度弱 | 后半场容易被压制 |

### 概率修正规则

| 情况 | 修正方向 |
|---|---|
| 强队缺少核心中场 | 胜率下调，平局上调 |
| 强队缺少主力中卫 / 后腰 | 对手进球概率上调 |
| 弱队主力门将状态极佳 | 爆冷和平局概率上调 |
| 弱队缺少反击核心 | 爆冷概率下调 |
| 首发比预期保守 | 平局和小比分概率上调 |
| 首发比预期激进 | 胜负分出概率上调，平局下调 |

## 4. 战术相克与比赛形态

### 目的

足球不是简单的强弱比较，而是看比赛会被谁带进自己舒服的节奏。

### 核心问题

> **A 队最强的地方，能不能打到 B 队最弱的地方？**  
> **B 队有没有办法让 A 队踢得不舒服？**

### 主要变量

| 子变量 | 观察点 |
|---|---|
| **阵型对位** | 4-3-3、4-2-3-1、3-4-3、5-4-1 等 |
| **压迫强度** | 高位逼抢还是中低位防守 |
| **控球能力** | 能否稳定出球、控节奏 |
| **破密防能力** | 面对低位防守是否有办法 |
| **反击速度** | 边锋、中锋、前腰是否有冲击力 |
| **边路强弱** | 是否能打穿对方边后卫身后 |
| **中场对抗** | 技术、身体、二点球争夺 |
| **传中与高空球** | 是否有高点、是否能防高球 |
| **转换防守** | 丢球后能否迅速回防 |
| **落后后的变招** | 是否有 B 计划 |
| **领先后的守局能力** | 能否控制节奏、减少失误 |

### 常见战术相克

| A 队特点 | 容易被谁克制 |
|---|---|
| 高位压迫 | 长传能力强、前场速度快的反击队 |
| 控球传控 | 低位密防、身体强、定位球强的球队 |
| 三中卫体系 | 边路速度快、能打翼卫身后的球队 |
| 边路传中多 | 中卫高大、禁区保护好的球队 |
| 中路渗透强 | 后腰防守强、禁区前沿保护好的球队 |
| 反击型球队 | 控球稳定、丢球后反抢快的球队 |
| 技术型中场 | 身体对抗强、压迫凶的球队 |

### 加分信号

| 信号 | 说明 |
|---|---|
| 强队刚好能打对手弱点 | 胜率明显上调 |
| 弱队反击点正好针对强队慢中卫 | 爆冷概率上调 |
| 强队边路优势明显 | 进球概率上调 |
| 强队有多套进攻方案 | 不容易被单一限制 |
| 弱队低位防守纪律好 | 平局和小比分上调 |

### 扣分信号

| 信号 | 说明 |
|---|---|
| 强队只会一种打法 | 被限制后容易急躁 |
| 控球多但缺少纵深 | 破密防困难 |
| 高位压迫但中卫速度慢 | 容易被身后球打穿 |
| 边后卫压得太高 | 反击风险上升 |
| 后腰覆盖不足 | 二点球和禁区前沿危险 |

### 判断方式

| 比赛形态 | 概率影响 |
|---|---|
| 强队能持续压制并创造高质量机会 | 强队胜率上调 |
| 强队控球多但机会少 | 平局概率上调 |
| 弱队有清晰反击路线 | 爆冷概率上调 |
| 双方中场缠斗严重 | 小比分概率上调 |
| 双方转换都快 | 大比分和胜负分出概率上调 |

## 5. 赛程、体能、旅行、气候

### 目的

世界杯不是单场孤立比赛。旅行、休息、温度、海拔和消耗会明显影响强度，尤其影响高压逼抢队和老龄化球队。

### 主要变量

| 子变量 | 观察点 |
|---|---|
| **休息天数** | 是否比对手少休息 1-2 天 |
| **上一场消耗** | 是否经历高强度、加时、点球大战 |
| **旅行距离** | 是否跨城市、跨时区长途移动 |
| **气候条件** | 高温、湿度、低温、风雨 |
| **海拔适应** | 高海拔会影响冲刺和恢复 |
| **开球时间** | 午后高温比晚场更耗体能 |
| **阵容年龄结构** | 老将多的球队恢复更慢 |
| **轮换能力** | 替补是否能维持强度 |
| **赛程密度** | 小组赛三场间隔，淘汰赛恢复时间 |
| **训练基地距离** | 是否频繁移动 |

### 加分信号

| 信号 | 说明 |
|---|---|
| 休息时间更长 | 后半场优势明显 |
| 阵容年轻且轮换深 | 高强度可持续 |
| 已适应当地气候 | 环境负面影响小 |
| 上一场轻松取胜 | 体能保存较好 |
| 无长途旅行 | 备战更稳定 |

### 扣分信号

| 信号 | 说明 |
|---|---|
| 少休息 2 天以上 | 后 30 分钟风险上升 |
| 上一场加时 / 点球 | 体能和心理消耗大 |
| 高温下打高位压迫 | 压迫强度可能下降 |
| 老将多 | 后半场容易掉速 |
| 连续跨城市旅行 | 恢复和训练质量下降 |
| 海拔不适应 | 冲刺、回追、压迫受影响 |

### 判断方式

| 情况 | 概率影响 |
|---|---|
| 强队体能明显吃亏 | 胜率下调，平局上调 |
| 弱队体能更好且能反击 | 爆冷概率上调 |
| 高温 + 慢节奏 | 小比分和平局上调 |
| 双方都疲劳 | 失误、定位球、后程进球概率上调 |
| 一方替补深度明显更好 | 后 30 分钟进球概率上调 |

## 6. 定位球与门将

### 目的

世界杯单场比赛里，定位球和门将经常能打破实力差距。弱队想爆冷，最现实的路径通常是：**低位防守 + 门将发挥 + 定位球偷一个。**

### 定位球变量

| 子变量 | 观察点 |
|---|---|
| **角球质量** | 罚球落点、战术设计 |
| **任意球质量** | 直接任意球和间接任意球威胁 |
| **高点数量** | 中卫、中锋、后腰的头球能力 |
| **二点球能力** | 争顶后第二落点控制 |
| **界外球战术** | 长界外球、禁区混战 |
| **防定位球能力** | 盯人、区域、防守纪律 |
| **门将出击能力** | 能否控制高空球 |
| **禁区犯规风险** | 拉拽、手球、冒失铲球 |

### 门将变量

| 子变量 | 观察点 |
|---|---|
| **扑救能力** | 面对高质量射门的表现 |
| **出击稳定性** | 高球、单刀、身后球处理 |
| **脚下出球** | 能否帮助球队破压迫 |
| **心理稳定性** | 大赛压力下是否容易失误 |
| **点球能力** | 淘汰赛价值极高 |
| **长传发动反击** | 弱队反击的重要起点 |

### 加分信号

| 信号 | 说明 |
|---|---|
| 弱队定位球强 | 爆冷概率上调 |
| 强队防定位球差 | 丢球风险上调 |
| 门将状态好 | 平局和小比分概率上调 |
| 高点优势明显 | 角球、任意球威胁增加 |
| 点球门将强 | 淘汰赛里额外加分 |

### 扣分信号

| 信号 | 说明 |
|---|---|
| 门将出击不稳 | 高球和传中风险大 |
| 防定位球盯人混乱 | 容易被弱队偷一个 |
| 后卫禁区动作大 | 点球风险上升 |
| 门将脚下差 | 面对高压容易送机会 |
| 没有高点 | 落后时缺少简单直接方案 |

### 判断方式

| 情况 | 概率影响 |
|---|---|
| 弱队定位球优势明显 | 爆冷概率上调 |
| 强队门将明显更强 | 强队容错率上调 |
| 双方运动战差距不大，但一方定位球强 | 该队胜率小幅上调 |
| 淘汰赛门将点球能力强 | 点球大战概率下该队晋级概率上调 |

## 7. 爆冷触发器

### 目的

这是单独维度。爆冷不是简单地猜弱队赢，而是判断弱队有没有办法把比赛拖进强队不舒服的区间。

> **爆冷的本质：弱队把比赛变成低比分、低节奏、高对抗、高偶然性的比赛。**

### 爆冷触发器清单

| 爆冷触发器 | 具体表现 |
|---|---|
| **强队进攻依赖单点** | 只靠一个核心组织者或一个边路爆点 |
| **强队破密防能力差** | 控球很多，但禁区内机会少 |
| **强队中卫速度慢** | 容易被弱队反击打身后 |
| **强队后腰保护差** | 禁区前沿容易被打穿 |
| **强队边后卫压太高** | 身后空间巨大 |
| **强队体能差** | 高温、少休息、连续高强度比赛 |
| **强队已出线或平局够用** | 动机下降，轮换可能增加 |
| **弱队低位防守质量高** | 能守禁区，能压缩空间 |
| **弱队门将状态好** | 能把比赛拖进 0-0、1-1 |
| **弱队定位球强** | 角球、任意球、长传冲吊有威胁 |
| **弱队反击速度快** | 少数机会就能制造单刀或点球 |
| **弱队身体对抗强** | 能打乱强队节奏 |
| **天气场地不利技术流** | 高温、湿滑、草皮慢 |
| **裁判尺度偏宽** | 对身体强、低位防守队有利 |
| **早期红牌 / 点球 / 乌龙风险高** | 单个事件极大改变比赛 |

### 爆冷等级

| 爆冷等级 | 触发条件 | 判断 |
|---|---|---|
| **低爆冷风险** | 0-2 个触发器 | 强队优势较稳 |
| **中低爆冷风险** | 3-4 个触发器 | 强队仍占优，但不宜判断为稳胜 |
| **中高爆冷风险** | 5-6 个触发器 | 平局和小胜小负概率明显上升 |
| **高爆冷风险** | 7 个以上触发器 | 强队胜率要明显下调，弱队不败值得重视 |

### 爆冷概率修正规则

| 爆冷等级 | 强队胜率修正 | 平局修正 | 弱队胜率修正 |
|---|---:|---:|---:|
| 低爆冷风险 | 0 到 -2% | 0 到 +1% | 0 到 +1% |
| 中低爆冷风险 | -2% 到 -5% | +1% 到 +3% | +1% 到 +2% |
| 中高爆冷风险 | -5% 到 -10% | +2% 到 +5% | +3% 到 +5% |
| 高爆冷风险 | -10% 到 -15% | +4% 到 +7% | +5% 到 +8% |

### 常见爆冷比分

| 比赛走势 | 常见比分 |
|---|---|
| 弱队死守成功 | 0-0 |
| 强队先进但被定位球扳平 | 1-1 |
| 弱队定位球偷一个 | 0-1 |
| 弱队反击偷一个 | 0-1 / 1-2 |
| 强队久攻不下后被反击 | 0-1 |
| 强队领先后松动 | 1-1 / 1-2 |
| 强队轮换过大 | 1-1 / 0-1 |

## 8. 小组积分形势 / 比赛动机

### 目的

世界杯小组赛和淘汰赛的预测逻辑不同。尤其小组赛第三轮，不能只看实力，必须看晋级条件。

### 主要变量

| 子变量 | 观察点 |
|---|---|
| **是否必须赢** | 必须赢的队会更主动，也更容易暴露空间 |
| **平局是否够用** | 平局够用的队可能踢得保守 |
| **是否已经出线** | 可能轮换或降低强度 |
| **是否已经出局** | 动机不确定，有时放松反而踢得更好 |
| **净胜球需求** | 是否需要大胜 |
| **黄牌停赛风险** | 主力是否会避免吃牌 |
| **淘汰赛策略** | 90 分钟不输比冒险赢球更重要 |
| **对阵潜在对手** | 有时排名会影响淘汰赛路径 |
| **主帅风格** | 保守型还是激进型 |
| **国家舆论压力** | 热门队压力越大，久攻不下越容易急躁 |

### 场景判断

| 场景 | 概率影响 |
|---|---|
| 强队平局即可 | 胜率下调，平局上调 |
| 弱队必须赢 | 比赛开放度上升，双方进球概率上调 |
| 双方都能接受平局 | 低比分和平局概率明显上调 |
| 强队已经提前出线 | 轮换概率上升，胜率下调 |
| 弱队出局无压力 | 爆冷风险小幅上调 |
| 淘汰赛 90 分钟 | 平局概率通常高于小组赛 |
| 加时和点球可能性高 | 门将、点球手、替补深度权重上升 |

---

## 9. 裁判尺度 / 红牌点球变量

### 目的

裁判不是主要预测因素，但在低比分比赛中，一个点球或红牌可能直接改变结果。

### 主要变量

| 子变量 | 观察点 |
|---|---|
| **判罚尺度** | 偏严还是偏宽 |
| **黄牌频率** | 是否容易早早出牌 |
| **红牌倾向** | 高对抗队风险更大 |
| **点球倾向** | 禁区身体接触多的队风险上升 |
| **VAR 介入风格** | 手球、越位、禁区犯规 |
| **补时时长** | 后程进球概率 |
| **对抗尺度适配** | 技术队还是身体队受益 |
| **情绪管理** | 球队是否容易急躁、围裁判、报复犯规 |

### 加分信号

| 信号 | 说明 |
|---|---|
| 裁判尺度宽，弱队身体强 | 弱队防守更容易成立 |
| 裁判尺度严，强队盘带多 | 强队更容易获得任意球和点球 |
| 技术型球队面对高犯规队 | 点球和定位球机会可能增加 |
| 门将和后卫动作干净 | 降低送点风险 |

### 扣分信号

| 信号 | 说明 |
|---|---|
| 后卫动作大 | 点球风险高 |
| 中场拦截靠犯规 | 早牌后防守强度下降 |
| 情绪化球员多 | 红牌风险上升 |
| 高压逼抢动作大 | 被判犯规和吃牌概率增加 |
| 禁区防守习惯差 | VAR 下容易出问题 |

### 判断方式

| 情况 | 概率影响 |
|---|---|
| 裁判严格 + 一方盘带强 | 该队点球 / 定位球概率上调 |
| 裁判宽松 + 一方身体强 | 低比分和爆冷概率上调 |
| 一方防守动作大 | 红牌、点球风险上调 |
| 淘汰赛 + 严格裁判 | 比赛节奏可能被切碎，平局概率上调 |

# 三、综合变数池

除了 9 个主维度，还要把所有变量归入 5 类综合变数，方便最终判断。

## 1. 稳定性变数

| 变量 | 影响 |
|---|---|
| 主帅体系稳定 | 提高预测可信度 |
| 首发框架稳定 | 降低比赛波动 |
| 中轴线完整 | 提高强队兑现能力 |
| 近期表现稳定 | 降低爆冷概率 |
| 更衣室稳定 | 避免非技战术风险 |

**判断：**稳定性越高，强队越容易兑现实力；稳定性越低，爆冷和平局概率越高。

## 2. 偶然性变数

| 变量 | 影响 |
|---|---|
| 低比分环境 | 增加平局和爆冷 |
| 定位球优势 | 增加弱队偷球机会 |
| 门将超神可能 | 降低强队进球概率 |
| 红牌 / 点球风险 | 增加比赛随机性 |
| 天气和场地 | 削弱技术优势 |
| 强队久攻不下 | 心态急躁，失误增加 |

**判断：**偶然性越高，预测越不能给出过高胜率。即使强队明显占优，也要保留平局和爆冷空间。

## 3. 节奏变数

| 变量 | 影响 |
|---|---|
| 高压逼抢 | 提高比赛强度 |
| 低位防守 | 降低进球数 |
| 双方都反击快 | 增加大比分可能 |
| 一方平局够用 | 降低节奏 |
| 高温 / 疲劳 | 降低压迫强度 |
| 早早进球 | 改变原有节奏 |

| 节奏 | 倾向 |
|---|---|
| 慢节奏 | 0-0、1-0、1-1 |
| 快节奏 | 2-1、2-2、3-1 |
| 一方控球一方低防 | 1-0、1-1、2-0 |
| 双方都必须赢 | 2-1、2-2、3-2 |

## 4. 心理变数

| 变量 | 影响 |
|---|---|
| 热门压力 | 久攻不下容易急躁 |
| 弱队无压力 | 反而可能发挥更好 |
| 必须赢 | 进攻更积极，但防守风险更大 |
| 平局即可 | 心态更稳，但进攻欲望下降 |
| 大赛经验 | 关键时刻更冷静 |
| 点球大战经验 | 淘汰赛价值高 |

**判断：**心理优势不一定直接提高胜率，但会影响关键时刻的失误率和决策质量。

## 5. 信息变数

| 变量 | 影响 |
|---|---|
| 首发未公布 | 预测置信度下降 |
| 伤病信息不透明 | 风险上升 |
| 市场赔率突然变化 | 可能有隐藏信息 |
| 主帅赛前发言 | 可能暗示轮换或保守策略 |
| 训练情况 | 影响球员状态判断 |
| 天气临场变化 | 影响节奏和体能 |

**判断：**信息越不完整，最终预测越要降低置信度。首发公布后必须重新校正一次。

# 四、概率修正框架

## 1. 基础流程

```markdown
第一步：用基础实力 / Elo / 市场概率给出初始胜平负
第二步：用 xG、xGA、射门质量修正真实攻防状态
第三步：用阵容伤停和首发质量修正关键位置影响
第四步：用战术相克判断比赛会进入谁的节奏
第五步：用赛程、体能、旅行、气候修正后程强度
第六步：用定位球和门将判断低比分变数
第七步：单独检查爆冷触发器
第八步：结合小组积分形势和比赛动机
第九步：加入裁判尺度、红牌、点球等极端变量
第十步：输出胜平负概率、比分区间、爆冷路径和最终倾向
```

## 2. 概率调整原则

| 情况 | 修正方向 |
|---|---|
| 强队实力优势大，数据也支持 | 强队胜率上调 |
| 强队实力强，但 xG 不好 | 强队胜率下调 |
| 强队控球强，但破密防差 | 平局和小比分上调 |
| 弱队防守纪律好 | 平局上调 |
| 弱队定位球强 | 爆冷上调 |
| 弱队门将状态好 | 平局和爆冷上调 |
| 强队伤停集中在中后场 | 对手进球概率上调 |
| 强队缺少核心组织者 | 胜率下调 |
| 强队体能差 | 后程丢球概率上调 |
| 双方都必须赢 | 胜负分出概率上调 |
| 双方都接受平局 | 平局概率大幅上调 |
| 淘汰赛 | 90 分钟平局概率上调 |
| 裁判严 + 盘带强队 | 点球和定位球概率上调 |
| 裁判宽 + 身体队 | 爆冷和低比分概率上调 |

```markdown
- 中卫速度慢

```

# 五、比分判断模型

## 1. 低比分条件

满足越多，越倾向 0-0、1-0、1-1。

| 条件 |
|---|
| 一方或双方平局可以接受 |
| 淘汰赛第一阶段谨慎 |
| 强队破密防能力一般 |
| 弱队低位防守纪律好 |
| 双方 xG 都不高 |
| 高温、疲劳、节奏慢 |
| 门将状态好 |
| 裁判吹罚频繁，比赛被切碎 |
| 双方中场缠斗严重 |
| 强队缺少终结点 |

常见比分：`0-0`、`1-0`、`1-1`、`2-0`、`0-1`

## 2. 中高比分条件

满足越多，越倾向 2-1、2-2、3-1。

| 条件 |
|---|
| 双方都必须赢 |
| 双方防线都有明显漏洞 |
| 一方高压，一方反击快 |
| 边路空间大 |
| 中卫速度慢 |
| 后腰保护差 |
| 门将不稳 |
| 早早进球可能性高 |
| 一方定位球强，另一方防空差 |
| 比赛后段体能会崩 |

常见比分：`2-1`、`2-2`、`3-1`、`1-2`、`3-2`

## 3. 强队稳胜条件

| 条件 |
|---|
| 强队基础实力明显高 |
| xG 优势明显 |
| 阵容完整 |
| 战术正好克制对手 |
| 弱队反击点不明显 |
| 弱队定位球威胁一般 |
| 强队体能无明显劣势 |
| 强队必须赢且动机强 |

常见比分：`2-0`、`2-1`、`3-0`、`3-1`、`1-0`

## 4. 爆冷比分条件

| 条件 |
|---|
| 弱队低位防守好 |
| 弱队门将状态好 |
| 强队进攻依赖单点 |
| 强队久攻不下 |
| 弱队定位球强 |
| 强队后防速度慢 |
| 强队体能或动机有问题 |
| 裁判尺度利于身体对抗 |
| 比赛节奏被拖慢 |

# 十、产品化冻结设计 v4.1：AI 发现源 + 固化公开源抓取

## 1. 核心定位

预测终端不依赖单一付费 API。系统采用：

```markdown
AI 负责发现和判断公开数据源
脚本负责抓取已固化、已确认的公开数据源
模型负责计算底盘和情景结果
人工分析负责临场信息修正
```

脚本不能像 AI 一样全网乱搜。脚本只读取 `data/sources.json` 中已确认的来源，并将抓取结果写入赔率快照。

## 2. 数据源分层

| 层级 | 来源 | 用途 | 处理方式 |
|---|---|---|---|
| A | 手动赔率 / 本地配置 | 兜底、测试、稳定运行 | 始终可用 |
| B | 公开 HTML 页面 | 无 key 赔率抓取 | 正则适配，失败降级 |
| C | 公开 JSON | 无 key 结构化赔率 | 点路径适配，优先级高于 HTML |
| D | 需要 key 的 API | 可选增强 | 不作为默认依赖 |

## 3. 公开源准入规则

允许：

```markdown
无需登录
无需验证码
无需破解签名
无需绕过加密接口
页面或公开 JSON 能直接访问
可限频、可缓存、失败可降级
```

不做：

```markdown
绕登录
绕验证码
绕反爬
伪造加密客户端签名
抓账号后台或受限数据
```

## 4. 源配置

来源固化在：

```text
data/sources.json
```

HTML 正则源：

```json
{
  "source_id": "example_html",
  "type": "public_html_regex",
  "enabled": true,
  "url": "https://example.com/match",
  "patterns": {
    "home": "home-odds\">([0-9.]+)",
    "draw": "draw-odds\">([0-9.]+)",
    "away": "away-odds\">([0-9.]+)"
  }
}
```

JSON 点路径源：

```json
{
  "source_id": "example_json",
  "type": "public_json_path",
  "enabled": true,
  "url": "https://example.com/match.json",
  "paths": {
    "home": "odds.home",
    "draw": "odds.draw",
    "away": "odds.away"
  }
}
```

## 5. 源健康状态

每次刷新都记录：

```text
source_id
success_count
failure_count
last_success_at
last_failure_at
last_error
last_rows
```

健康状态保存到：

```text
data/source_health.json
```

总览页必须展示数据源状态，避免用户误以为赔率一定来自真实公开源。若所有公开源失败，系统必须自动降级到 `manual_odds`。

## 6. 预测闭环

```markdown
用户输入比赛
→ AI 查找候选公开源
→ AI 判断是否适合抓取
→ 写入 sources.json
→ 脚本刷新赔率快照
→ 面板展示市场变化、源健康、六情景预测
→ AI 读取脚本结果并联网补首发/伤停/天气/裁判
→ 输出最终分析
```

## 7. 产品要求

- 每场比赛必须绑定唯一 `match_id`。
- 赔率快照必须保留 `source`、`bookmaker`、`market`、`selection`、`odds_decimal`、`captured_at`。
- 模型/市场分歧必须基于最新可用快照。
- 数据完整度必须受源成功率影响。
- 公开源失败不得中断预测，应显示失败原因并降级。
- AI 发现源后的第一轮抓取结果必须人工确认后再长期启用。

# 十二、算法优化 v4.3

## 1. 动态市场权重

市场校准不再使用固定权重。市场权重由以下因素决定：

```markdown
bookmaker 数量
赔率快照数量
赔率波动强度
数据源健康状态
```

原则：

```markdown
来源越多，市场权重越高。
快照越多，市场权重越高。
多公司同向且波动明显，市场权重上调。
来源少或公开源失败多，市场权重下调。
```

## 2. 爆冷强度评分

爆冷触发器不再只数数量，改为：

```markdown
爆冷分 = 基础权重 × 强度 × 可信度
```

示例：

```json
{
  "underdog_low_block": {"strength": 1.2, "confidence": 0.8},
  "strong_low_block_problem": true
}
```

强队破密防差、核心缺阵、体能劣势权重更高；裁判和天气属于辅助变量，权重较低。

## 3. 比分分布修正

普通 Poisson 容易低估世界杯中的低比分和平局，因此按情景修正：

| 情景 | 修正 |
|---|---|
| 保守节奏 | 上调 0-0、1-1、1-0、0-1 |
| 开放节奏 | 上调 2-1、1-2、2-2、3球以上 |
| 爆冷情景 | 上调 0-0、1-1、0-1、1-2 |

## 4. 校准分桶

回测必须按置信区间查看：

```markdown
0-40%
40-50%
50-60%
60-70%
70%+
```

如果 70%+ 桶实际命中率明显不足，说明模型过度自信，需要降低置信度上限或加重风险惩罚。

## 5. 自动调参建议

系统根据回测输出建议：

```markdown
市场模型优于基准 → 提高市场权重上限
市场模型弱于基准 → 检查赔率源或降低市场权重
爆冷模型 Top2 覆盖高 → 保留强爆冷修正
保守模型在低节奏场更好 → 提高平局和低比分修正
Log Loss 偏高 → 降低高置信预测上限
```

# 十三、总进球与大小球模块 v4.4

## 1. 规则原则

总进球数和大小球盘口必须分开输出：

```markdown
总进球数 = 预测比赛总进球的分布，例如 0球、1球、2球、3球、4球、5+球。
大小球盘口 = 针对 1.5 / 2.5 / 3.5 等盘口线计算大球和小球概率。
```

不能把“2-3球区间”和“小2.5/小3.5”混为一谈。

## 2. 总进球数

从比分矩阵直接求和：

```markdown
P(总进球 = n) = sum(home_goals + away_goals = n)
P(0-1球) = P(0球) + P(1球)
P(2-3球) = P(2球) + P(3球)
P(4+球) = P(4球及以上)
```

输出：

```markdown
最可能总进球
推荐区间
低比分风险
大比分风险
0/1/2/3/4/5+ 球概率
```

## 3. 大小球盘口

第一版固定计算：

```markdown
大/小1.5
大/小2.5
大/小3.5
```

计算：

```markdown
P(大1.5) = P(总进球 >= 2)
P(小1.5) = P(总进球 <= 1)
P(大2.5) = P(总进球 >= 3)
P(小2.5) = P(总进球 <= 2)
P(大3.5) = P(总进球 >= 4)
P(小3.5) = P(总进球 <= 3)
```

倾向阈值：

```markdown
P(小3.5) >= 70%：小3.5优先
P(小2.5) >= 56%：小2.5可选
P(大2.5) >= 54%：大2.5可选
P(大1.5) >= 62%：大1.5可选
```

## 4. 情景修正

```markdown
保守节奏：0球、1球、2球上调；小2.5、小3.5上调。
开放节奏：3球、4球、5+球上调；大2.5上调。
爆冷情景：0-0、1-1、0-1、1-0上调；小球和低比分上调。
双方必须赢：2-3球和4+球上调。
一方平局可接受：0-1球和小球上调。
```

# 十四、盘口价值与体彩执行层 v5.0

本节作为 `盘口预测算法优化设计 v2.0` 与 `体彩下注建议页 v2.0-TC` 的落地规则。它不替代足球判断维度，只规定下注价值如何计算。

## 1. 底层计算链

```markdown
赔率快照
→ 隐含概率 1 / odds
→ 同市场去水概率
→ ln(市场概率) + 基本面/临场 Δ
→ softmax 归一化
→ 模型概率
→ 公允SP = 1 / 模型概率
→ EV = 模型概率 × 当前SP - 1
→ Kelly = (当前SP × 模型概率 - 1) / (当前SP - 1)
→ 风险分与仓位封顶
```

核心原则：

```markdown
方向判断看模型概率。
下注价值看 EV。
下注多少看 Kelly 折扣和风险上限。
长期能力看 CLV、ROI、Brier Score、Log Loss。
```

## 2. 基本面修正

基本面和临场信息只做小幅 log 修正，不直接推翻市场底盘。

默认系数：

| 维度 | 系数 |
|---|---:|
| 基础实力 | 0.040 |
| 近期过程质量 | 0.030 |
| 进攻/防守质量 | 0.030 |
| 阵容首发 | 0.040 |
| 战术相克 | 0.030 |
| 节奏 | 0.020 |
| 定位球/门将 | 0.020 |
| 体能环境 | 0.020 |
| 动机 | 0.015 |
| 裁判变量 | 0.015 |
| 爆冷触发器 | 0.030-0.040 |

如果模型概率与市场去水概率差异超过 15%，必须人工复核数据源、赔率录入、首发和伤停。

## 3. 体彩玩法映射

竞彩足球口径固定为：

```markdown
90分钟 + 伤停补时，不含加时赛，不含点球大战。
```

玩法映射：

| 玩法 | 概率来源 |
|---|---|
| 胜平负 | softmax 的主胜/平/客胜 |
| 让球胜平负 | Poisson 比分矩阵，`G_home + H - G_away` |
| 比分 | 官方31项，含胜其它/平其它/负其它 |
| 总进球 | 比分矩阵汇总为 0/1/2/3/4/5/6/7+ |
| 半全场 | 半场/全场联合分布，默认半场进球占比 0.45 |

官方开售状态校验：

中国体彩竞彩足球赛程页 `https://www.lottery.gov.cn/jc/zqszsc/` 通过官方接口 `getMatchListV1.qry?clientCode=3001` 提供：

- 比赛编号；
- 比赛开始时间；
- 玩法池 `HAD/HHAD/CRS/TTG/HAFU`；
- `cbtValue` 是否开售；
- `cbtSingle` 是否支持单关；
- `cbtAllUp` 是否支持过关；
- `goalLine` 让球数；
- `remark` 特别提示。

推荐规则必须先通过开售校验：

1. 未开售或暂停销售的玩法不输出下单建议。
2. `cbtSingle=0` 的玩法不能输出单关建议。
3. `cbtAllUp=0` 的玩法不能进入串关池。
4. 让球胜平负必须使用官方 `goalLine`，不能用模型估计盘口。
5. 特别提示、比赛场地和开售状态变化要进入风险备注。

比分展示修正：

| 层级 | 比分范围 | 用法 |
|---|---|---|
| 常见比分 | 0-0、1-0、0-1、1-1、2-0、0-2、2-1、1-2、2-2 | 主报告优先展示 |
| 扩大比分 | 3-0、0-3、3-1、1-3、3-2、2-3 或总进球4且分差不大 | 节奏打开时小额观察 |
| 冷门比分 | 高总进球、大分差、低概率精确比分 | 只观察，不进核心 |
| 其它比分 | 胜其它、平其它、负其它 | 范围太宽，只能作为冷门提示 |

比分是高方差玩法，不能直接按最高 EV 或最高奖金排序。`胜其它/平其它/负其它` 即使因为体彩奖金高导致 EV 看起来好，也不能排在比分主栏前面。

赛后复盘新增修正（2026-06-15）：

本轮德国 7-1 库拉索、荷兰 2-2 日本、科特迪瓦 1-0 厄瓜多尔暴露了三个偏差：

1. 深盘强弱悬殊场不能继续套低比分模板。德国让3球且官方让球市场更支持打穿时，模型仍过度展示 `2-0/1-0/2-1`，低估了 `4-0/5-0/胜其它/总进球7+` 的尾部风险。
2. 让球胜平负不能只用 Poisson 比分矩阵。若模型概率和官方SP去水概率分歧很大，应提高官方盘口权重，并降级模型单边结论。
3. 比分串关不能只走单一路径。荷兰 2-2 日本说明实力接近场若允许比分，应使用 `主比分+保护比分` 的复式，而不是只串 `1-1 × 1-1 × 1-0`。

修正规则：

| 场景 | 触发条件 | 修正 |
|---|---|---|
| 深盘强弱悬殊 | 官方让球绝对值≥2，且 Elo差≥300 / 胜负概率差≥50% / xG差≥1.75 任一成立 | 比分参考切换到强队大胜区间，展示大胜尾部；低总进球高赔降级 |
| 盘口-模型冲突 | 模型概率与官方SP去水概率差异≥16个百分点 | 该选项不能作为核心腿，最多观察 |
| 深盘让球 | 官方让球绝对值≥2 | 让球胜平负概率中官方SP去水概率权重提高到约65% |
| 普通让球 | 官方让球绝对值=1 | 官方SP去水概率权重约42%，同时保留比分矩阵映射 |
| 比分复式 | 实力接近、平局空间足、xG处于中低区间 | 比分可用 `主比分+保护比分` 小复式，但预算低于方向主仓 |
| 开放尾部 | 赛前存在强队多终结点、定位球/空中球优势、弱队落后后防线拉开、弱队追分前压、弱队防守样本强度不足、后段体能差等信号 | 上调 `2-1/3-1/3-0/2-2/3-2/4-1` 和总进球4+；极端比分只进尾部池 |

瑞典 vs 突尼斯复盘口径：

- 不能因为赛果直接提高开放节奏权重。
- 赛前可识别偏差是：瑞典锋线多终结点和定位球优势被低估，突尼斯预选赛防守数据样本强度需要折扣，突尼斯落后后追分导致防线被拉开的尾部风险没有进入主比分池。
- 赛后才知道的信息，例如具体进球时间、最后阶段崩盘、补时继续进球，不能倒灌成赛前权重。
- 正确修正是增加 `open_game_triggers` 条件特征，触发后扩大比分池，而不是长期上调所有比赛的大比分概率。

计算修正：

1. 比分矩阵使用温和 Dixon-Coles 低比分相关性修正，避免独立 Poisson 低估 0-0、1-1。
2. 基准/市场场景只做小幅修正；保守/爆冷场景加强低比分相关性；开放场景弱化修正。
3. `胜其它/平其它/负其它` 从比分矩阵自身求和，不再用胜平负概率直接减去已列比分。
4. 胜平负概率只对比分矩阵做温和校准，防止市场校准差异全部挤到“其它比分”。
5. 比分风险分随命中率和层级上调：其它比分、高总进球/大分差比分、低命中率比分只作观察。

实力接近场的比分规则：

- Elo差≤180；
- 主客胜概率差≤22%；
- 平局概率≥24%；
- 双方 xG 都在 0.75-1.90；
- 主比分集中在常见低比分或一球小胜区间。

执行要求：

- 优先比分：`0-0`、`1-1`、`1-0`、`0-1`、`2-1`、`1-2`；
- 比分预算不超过总资金 5%-10%；
- 比分不作为核心串关腿；
- 若盘口为受让1且主比分含 `0-1`，让球胜平负必须优先检查 `让平`；
- 若盘口为让1且主比分含 `1-0`，让球胜平负必须优先检查 `让平`，并防强队不穿盘。

| 半全场 | 10% | 80 | 0.2% |

动作：

```markdown
SP缺失：不可用
EV <= 0：放弃
0 < EV < 玩法门槛：观察
EV 达门槛但风险超限：高风险观察
EV 达门槛且风险可控：可小注
```

## 5. 复选包与串关

复选包按注数重算：

```markdown
复选包EV = SUM(p_i × SP_i) / 注数 - 1
```

串关只从正 EV 候选池生成：

```markdown
组合概率 = PRODUCT(p_i)
组合SP = PRODUCT(SP_i)
组合EV = 组合概率 × 组合SP - 1
```

串关红线：

```markdown
每一腿 EV 必须 > 0。
组合 EV 必须 >= 5%。
同一场不同玩法不得混入同一串。
低赔但负EV热门不进串关。
比分和半全场不作为核心串关腿。
串关总金额不超过本金池 0.5%。
```

# 十七、v3 分布与串关风险算法

## 1. λ 加权构建器

`expected_goals` 不再只代表静态 xG，而是基础 xG 经过正式赛强度、阵容、赛程、战术、定位球、动机和节奏后的最终输入值。

模型必须输出 `lambda_adjustments`，说明本场进球预期为何上调或下调。

## 2. 混合比分分布

| 场景 | 分布策略 |
|---|---|
| 普通场 | Poisson + Dixon-Coles 低比分修正 |
| 开放局 / 爆冷触发 | Poisson + Negative Binomial 混合，放大比分方差 |
| 深盘强弱场 | 保留大胜尾部，比分展示优先进入 3球以上胜区 |
| 低节奏场 | 压缩 4球以上尾部，提高 0-1球区间 |

## 3. 一球小胜偏移

只在让 `-1/+1` 的小胜格局启用：

```text
胜率有优势，但 xG差不夸张；
总进球不高；
盘口不明显支持大胜；
主比分落在 1:0 / 0:1 / 2:1 / 1:2 一带。
```

深盘场禁止全局套用一球偏移，避免和让胜判断冲突。

## 4. 让球区间映射

体彩让球按主队让球后的比分计算：

```text
adjusted_diff = home_goals + handicap - away_goals
adjusted_diff > 0：让胜
adjusted_diff = 0：让平
adjusted_diff < 0：让负
```

## 5. 串关结构风险

串关不是只做概率乘法，还必须识别风险集中度：

```text
favorite_cover：热门穿盘共振风险
one_goal_margin：多场一球小胜共振风险
favorite_not_cover：热门不穿盘共振风险
high_leg：多个高风险腿叠加
```

样本不足时，相关性只能用于风险扣分，不能用于盲目提高联合命中率。

# 十八、v5.5 首轮热门降温与平局保护

## 1. 优化目标

这次优化针对的不是某一场赛果，而是一个可复用的赛前偏差：强队深盘、大众热门、总进球预期不高、弱队低位防守时，模型容易把热门大胜和穿盘放得过重，低估 `0:0`、`1:1`、一球小胜和让球不穿盘。

## 2. 触发式规则

只有满足赛前信号才启用，不允许赛后按结果倒改：

| 信号 | 含义 | 处理 |
|---|---|---|
| 热门过热 | 概率差、xG差或让球较深 | 降低热门穿盘权重 |
| 总进球不高 | 合计xG偏低或节奏偏慢 | 提升低比分/平局保护 |
| 弱队低位防守 | 弱队有摆大巴、门将发挥、定位球守分路径 | 加入 `0:0/1:1/1球差` |
| 小组首轮/早段 | 强队可能先稳局面 | 禁止默认大胜 |
| 盘口过深 | 让2球及以上 | 大胜比分只做尾部保护 |

## 3. 模型调整

### 3.1 胜平负层

新增 `favorite_stall_profile`：

```text
score = 热门过热 + 深盘 + 低总进球 + 首轮 + 低位防守触发器
```

当 `score >= 0.45`：

- 热门方向小幅降温；
- 平局概率小幅上调；
- 不直接反推冷门胜，只先保护“平/不穿盘/低比分”。

### 3.2 比分层

比分矩阵新增两项修正：

- `draw_low_score_boost`：提升 `0:0`、`1:1`、低比分一球差；
- `favorite_cover_cooldown`：降低热门净胜3球以上的概率权重。

深盘热门触发保护后：

- `3:0/4:0/4:1/5:0/5:1` 不再自动前置；
- `0:0/1:1/1:0/2:0` 等进入保护池；
- 大胜比分输出时必须标记为“尾部防线”。

### 3.3 让球层

让球胜平负必须从比分矩阵映射：

```text
adjusted_diff = home_goals + handicap - away_goals
```

当热门降温触发：

- 让胜降低优先级；
- 让平、让负可作为防穿盘观察；
- 反主线项仍不得直接变成主推。

### 3.4 总进球层

深盘场不再机械否定低总进球：

- 未触发热门降温：低总进球高赔仍视作陷阱；
- 触发热门降温：`0/1/2球` 可作为小额保护，不进核心重仓。

## 4. 串关与仓位修正

新增串关主题暴露检查：

```text
theme:favorite_cover
theme:popular_winner
theme:draw_low_score_protection
theme:favorite_tail_hedge
theme:low_total_protection
```

规则：

- 3串、4串不能多腿重复依赖热门穿盘或热门胜。
- 多场热门同向时，只能标记高风险观察。
- 同一主题串关仓位必须受 `same_theme_combo_cap` 限制。
- 热门穿盘仓位受 `favorite_cover_cap` 限制。

## 5. 展示要求

页面和 Excel 必须用普通话解释：

- “热门不一定打穿，需要防平/小胜/低比分。”
- “大胜比分是尾部防线，不是主推。”
- “这组串关多腿都靠热门打穿，风险集中。”

不得使用“干净腿”等用户难理解的词。推荐必须写清：玩法、选择、SP、投入、理论返奖、命中条件、最大风险点。

# 二十一、v5.8 完赛数据回测与热门降温校准

## 1. 完赛数据接入原则

- 完赛结果优先使用稳定结构化源：项目内配置的中国体彩、爱彩世界杯公开数据、Bing/公开积分赛果源；MSN 只作为页面人工核验源，不作为默认抓取依赖。
- 完赛数据必须写入本地 `matches` 和 `fixtures`，字段至少包含：比赛ID、主队、客队、北京时间、状态、全场比分、半场比分、来源、更新时间。
- 赛后过程数据可进入球队状态修正：射正、射门、控球、犯规、黄红牌、首发阵型、球员技术统计。但只能影响后续比赛，不能倒填已归档赛前预测。

## 2. 赛前赛后匹配规则

- 回测必须优先用同一 `match_id` 匹配赛前预测快照。
- 若公开源刷新后 `match_id` 改变，允许用“主队+客队+比赛日期”生成别名匹配，把旧预测快照与新完赛赛果连起来。
- 同一场同一情景模型如果存在多次赛前快照，摘要统计只使用最新快照；原始快照数保留展示，避免同一场比赛重复放大回测结论。

## 3. 回测指标与调参边界

- 回测指标：Top1、Top2、比分Top1、Brier、Log Loss、ROI、赛前SP、收盘SP、CLV。
- 若 Top1 低但 Top2 高，说明方向判断偏激进，不应简单推翻模型；应把单选热门转为双选、防让平、低比分防护。
- 若连续出现平局漏判，重点提高这些场景的平局/让平权重：小组赛早期、强弱差不是极端、让球为1或2球、总进球预期不高、热门队破密防风险高。
- 若连续出现强队不穿盘，降低热门队置信上限，比分池增加 0-0、1-0、1-1、2-1 等低比分和一球小胜候选。
- 单场意外比分不得直接改变全局权重；只有同类偏差在多场回测中重复出现，才进入参数修正。

## 4. 本轮算法调整方向

- `favorite_stall_profile` 命中时，对热门方向做小幅降温。
- 平局增量从“普通防护”升级为“小组早期/深盘/低总进球”组合防护。
- 让球为 1 或 2 球时，额外加强让平与不穿盘提醒。
- 回测建议不直接生成下注结论，只作为下一轮预测和候选池排序的校准输入。
5. 体彩计算器下单视图：只显示官方可买玩法。

## 7. 实现状态

- 模型层函数：`side_signal_profiles`、`folk_parallel_summary`。
- 兼容旧字段：已保留。
- 输出位置：`prediction.sporttery.folk_parallel`。
- 页面展示：必须用普通话解释，不使用“干净腿”等难懂术语。

# 二十二、v5.9 行动分与决策闭环

v5.9 不改写主模型概率。它是在概率、EV、风险和体彩可买性之后增加的推荐执行层，用来解决“模型说了很多，但用户不知道怎么买、该不该买”的问题。

## 1. 行动分输入

| 输入 | 作用 |
|---|---|
| 模型概率 | 判断结果本身是否成立 |
| 官方SP / 去水概率 | 判断价格是否划算 |
| EV / 公允SP | 判断是否有正期望 |
| 风险分 | 控制爆冷、分歧、阵容和盘口波动 |
| 玩法类型 | 胜平负/让球/总进球更稳定，比分/半全场默认高波动 |
| 比分主线一致性 | 让球推荐必须能被比分池解释；反主线只能做观察 |
| 官方可买性 | 未开售、缺SP、玩法暂停时不可下单 |

## 2. 行动层级

| 层级 | 含义 | 仓位规则 |
|---|---|---|
| 主推 | 价值、风险和玩法稳定性较均衡 | 可进入首页下单清单核心项 |
| 可搭配 | 可做组合搭配，但置信度低于主推 | 金额不得高于主推 |
| 防冷小注 | 只覆盖爆冷、穿盘、低比分或尾部风险 | 小仓，不能当主仓 |
| 观察 | 有方向或价格参考，但条件不完整 | 不进下单清单 |
| 放弃 | EV<=0或风险收益不匹配 | 不下单 |
| 不可下单 | 体彩未开售、缺少SP或玩法不可买 | 不下单 |

## 3. 首页决策闭环

首页必须优先服务当天决策，不再只做赛程挂历：

```text
最新完赛复盘 -> 待开比赛 -> 今日下单清单 -> 当前预测 -> 回测提醒 -> 混合过关候选
```

今日下单清单只允许展示：

```text
有官方体彩SP
行动层级为主推 / 可搭配 / 防冷小注
EV为正或符合防冷小注规则
能写清玩法、选择、投入、理论返奖和风险说明
```

禁止把缺SP、负EV、不可买、反主线但未标注的项目放进首页下单清单。
